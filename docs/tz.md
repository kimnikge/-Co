## 5.1 Админ-панель

URL: `/admin`

Назначение:
Управление входящими заявками без использования внешней CRM.

Функционал:

- Авторизация по логину и паролю (ENV)
- Просмотр списка заявок
- Сортировка по дате
- Фильтр по статусу (new / in_progress / closed)
- Изменение статуса заявки
- Просмотр деталей заявки
- Экспорт CSV
- Логаут

Структура заявки:

{
  id: string,
  date: ISO string,
  name: string,
  phone: string,
  businessType: string,
  regime: string,
  status: "new" | "in_progress" | "closed"
}

Статусы:
- new — новая заявка
- in_progress — в работе
- closed — обработана

Безопасность:
- Доступ только через авторизацию
- Cookie httpOnly
- Rate limit на логин
- Логирование входов

## 14.4 GET /api/export




- monthlyRevenue (число)
- payroll (число)


### 1. УСН (примерная модель)

```
taxUSN = monthlyRevenue * usnRate
```

Где:
- Дополнительно учитываются соц. отчисления от payroll

---
corporateTax = grossProfit * corporateRate
vat = hasVAT ? monthlyRevenue * vatRate : 0
```

Общая нагрузка:
```
totalOUR = corporateTax + socialTax + vat
```

---

## 15.3 Сравнение режимов

```
difference = totalOUR - taxUSN
```

Если difference < 0 → ОУР выгоднее  
Если difference > 0 → УСН выгоднее  

---

## 15.4 Конфигурационные параметры

Вынести в отдельный файл:

`src/lib/config/taxConfig.ts`

Пример:

```ts
export const taxConfig = {
  usnRate: 0.03,
  corporateRate: 0.20,
  socialRate: 0.095,
  vatRate: 0.12
};
```

Все ставки должны задаваться централизованно.

# 16. Auth-flow админ-панели

## 16.1 Модель авторизации

Простая парольная авторизация через ENV.

ENV переменные:

```
ADMIN_LOGIN
ADMIN_PASSWORD
SESSION_SECRET
```

---

## 16.2 Процесс входа

1. Администратор заходит на `/admin`
2. Если нет cookie → отображается форма логина
3. После ввода:
   - Сервер сверяет логин/пароль
   - Генерирует подписанную cookie (httpOnly)
4. Пользователь получает доступ

---

## 16.3 Защита маршрутов

В `+layout.server.ts`:

- Проверка cookie
- Если отсутствует → redirect 302 на `/admin/login`

---

## 16.4 Безопасность

- Cookie: httpOnly
- secure: true (в production)
- SameSite: strict
- Ограничение количества попыток входа
- Логирование входов
